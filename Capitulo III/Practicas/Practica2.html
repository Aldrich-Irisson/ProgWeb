Ejercicios de autoevaluación

3.2 Indique si cada uno de los siguientes enunciados es verdadero o falso. Si es falso, explique por qué.


  1. Cualquier tipo Input particular de formulario de HTML5 debe desplegarse de manera idéntica en todos los navegadores compatibles con HTML5.

    2. R: Falso. El despliegue de los tipos input puede variar de un navegador a otro.


  2. Cuando se coloca el enfoque en el campo de texto (es decir, si el cursor está en el campo de texto ), se envía el texto del atributo placeholder al servidor.

    3. R: Falso. Cuando se coloca el enfoque en el campo de texto, el texto de placeholder desaparece. No se “envía” cuando el usuario hace clic en el botón Enviar (a menos que el usuario escriba el mismo texto).


  3. No es necesario incluir el atributo autofocus en sus formularios.

    4. R: Verdadero.


  4. Los nuevos tipos input de HTML5 tienen validación automática del lado del cliente, con lo cual se elimina la necesidad de agregar secuencias de comandos (script) complicadas a sus formularios para validar la entrada del usuario y se reduce la cantidad de datos inválidos que se envían.

    5. R: Verdadero.


  5. El tipo input range se valida automáticamente de manera intrínseca cuando el navegador lo despliega como un control deslizante, ya que el usuario no puede mover el control fuera de los límites del valor mínimo o máximo.

    6. R: Verdadero.


  6. HTML5 cuenta con validación automática para el tipo input tel.

    7. R: Falso.La longitud y el formato de los números telefónicos varía de manera considerable con base en la ubicación, por lo que la validación se vuelve bastante compleja, así que HTML5 no cuenta con validación automática para el tipo input tel. Para asegurar que el usuario introduzca el número en un formato apropiado, usamos el atributo pattern.


  7. Si el usuario introduce un URL con un formato inapropiado en un tipo input url, no se validará. HTML5 no válida si el URL introducido existe en realidad.

    8. R: Verdadero.


  8. El elemento nav muestra un menú desplegable de hipervínculos.

    9. R: Falso.El elemento nav agrupa vínculos de navegación.


  9. El elemento header puede usarse sólo una vez en una página.

    10. R: Falso.El elemento header puede usarse varias veces en una página y a menudo incluye encabezados de HTML (<hl> a <h6>).


  10. Los elementos nav pueden anidarse en un elemento aside.

    11. R: Verdadero.


  11. Podemos usar el elemento brk para evitar dividir palabras en lugares incorrecto.

    12. R: Falso. Puede usar el elemento wbr para evitar dividir las palabras en lugares incorrectos.
